Js闭包

闭包就是函数的局部变量集合，只是这些局部变量在函数返回后会继续存在。

闭包就是就是函数的“堆栈”在函数返回后并不释放，我们也可以理解为这些函数堆栈并不在栈上分配而是在堆上分配

当在一个函数内定义另外一个函数就会产生闭包

Sass

一、使用

1. 将.scss文件转化成css文件

命令行 sass test.scss (test.css)

2. 编译风格

  1. nested（默认值，嵌套缩进）
  2. expanded（没有缩进、扩展的css代码）
  3. compact（简洁格式）
  4. compressed（压缩）

命令行 sass --style compressed test.scss test.css

3. 监听文件变动并自动编译

file: sass --watch input.scss:output.css

directory: sass --watch app/sass:public/stylesheets

二、基本用法

1. 变量、计算、嵌套、注释

$blue: #1875e7;
$side: left;
$var: 14px;

div {
    //变量
    color: $blue;
    //变量镶嵌在字符串中就要写在#{}中
    border-#{$side}-radius: 5px;
    //计算
    margin: ($var/2);
    //嵌套
    h1 {
        color: red;
        //属性嵌套
        border: {
            color: red;
            width: 3px;
        }
    }
}
a {
    //&可引用父元素
    &:hover {
        color: $blue;
    }
}
/*!重要注释*/

2. 代码重用

2.1 继承
.class1 {
    border: 1px solid #ddd;
}
.class2 {
    @extend .class1;
    font-size: 120%;
}
2.2 Mixin
//@mixin命令定义一个可重用的代码块
@mixin left {
    float: left;
    margin-left: 10px;
}
//@include命令调用mixin定义的代码块
div {
    @include left;
}
//@mixin可指定参数和缺省值
@mixin left($value: 10px) {
    float: left;
    margin-right: $value;
}
div {
    @include left(20px);
}
//@mixin实例
@mixin rounded($vert, $horz, $radius: 10px) {
    -webkit-border-#{$vert}-#{$horz}-radius: $radius;
    -moz-border-radius-#{$vert}#{$horz}: $radius;
    border-#{$vert}-#{$horz}-radius: $radius;
}
#navbar li {
    @include rounded(top, left);
}
#footer {
    @include rounded(top, left, 5px);
}

3. 颜色函数

lighten(#cc3, 10%) //#d6d65c
darken(#cc3, 10%) // #a3a329
grayscale(#cc3) // #808080
complement(#cc3) // #33c

4. 插入文件

@import "path/filename.scss";
@import "foo.css";

三、高级用法

1. 条件语句

p {
    @if 1 + 1 == 2 { border: 1px solid; }
    @if 5 < 3 { border: 2px dotted; }
    @if lightness($color) > 30% {
        background-color: #000;
　　} @else {
　　      background-color: #fff;
　　}
}

2. 循环语句

$i: 6;
//2.1 for
@for $i from 1 to 10 {
    .border-#{$i} {
        border: #{$i}px solid blue;
    }
}
//2.2 while
@while $i > 0 {
    .item-#{$i} { width: 2em * $i };
    $i: $i - 2;
}
//2.3 each
@each $member in a, b, c, d {
    .#{$member} {
        background-image: url("/image/#{$member}.jpg");
    }
}

3. 自定义函数

@function double($n) {
    @return $n * 2;
}
#sidebar {
    width: double(5px);
}

Throttle vs. Debounce

Posted on 2016-10-24

两个函数都是为了降低回调函数触发的频率而存在的，但有一定区别。

###函数节流 Throttle

  • 原理：就是间隔一定的时间来触发回调以控制
  • 应用场景：
    1. 拖拽功能（mousemove）
    2. 射击游戏的 mousedown/keydown 事件（单位时间内只发射一次）
    3. 计算鼠标移动的距离（mousemove）
    4. canvas 模拟画板功能（mousemove）
    5. 搜索联想（keyup）
    6. 监听滚动事件判断是否到页面底部自动加载更多（给 scroll 加了 debounce 后，只有用户停止滚动后才会判断是否到了页面底部；如果是 throttle 的话，只要页面滚动就会间隔一段时间就判断一次）

###函数去抖 Debounce

  • 原理：对于一定段时间内连续的函数调用，之让其执行一次
  • 应用场景：
    1. 每次 resize/scroll 触发统计事件
    2. 文本输入的验证
